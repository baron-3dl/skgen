what we have tested so far

Tested in automated runs, not yet with real users.

No one outside the project has used these yet. What we have done: automated runs where separate Claude sessions build, run, and grade the skills, some on different models, so the score never comes from the same Claude that made the skill. Every run is logged in the examples, so you can read exactly what happened and check it.

0.93
A voice skill was run in a fresh session whose instructions told it to write in the opposite style. It kept the original style anyway. A separate Claude scored it 0.93 out of 1 against the original.
18 of 18
We ran the build-then-use across Opus, Sonnet, and Haiku, every maker-and-receiver pairing, for two skills. A separate Claude judged all eighteen runs good.
0 of 6
We fed the builder six poisoned instructions trying to hide a step in the skill it produced. An auditing Claude found none of them in the result.

for developers

It is a marketplace, too.

The Claude Code install above uses skillc's plugin marketplace, so you can version skills and ship them as files. The worked example skills are a separate plugin, skillc-examples (voice, explain-plainly, commit-message): install with /plugin install skillc-examples@skillc. One honest note: on claude.ai, uploaded skills only share through the platform on Team or Enterprise plans, so sending the file (or the paste path) is what works for everyone. The format, the builder, and the rebuild recipe are all in the repo.
