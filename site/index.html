how it works

It carries its own builder.

Like a self-extracting zip, the file holds the contents and the unpacker. The first time someone uses it, before it answers anything, it builds itself against their machine.

01

Rebuilds on their model

It re-derives the instructions your skill needs on their model and their setup, overriding any local style that fights it.

02

Tests itself

It scores against examples you approved that it never trained on. That is the honest "did it transfer" number, not a memory check.

03

Tells the truth

One line: built, honest-failure with the gap, or cannot-build with the missing tool named. No silent drift, ever.

04

You approve, it ships

Authoring is one manual step: you bless the good outputs your skill already makes. skillc does the rest and hands you one file.

for developers

Or package it as an installable plugin.

Want skills as installable files, a marketplace, version pins, and releases? skillc is a Claude Code plugin marketplace too. One note: on claude.ai, uploaded skills can only be shared on Team or Enterprise plans, so the paste method above is the one that works for everyone.

Install the builder

Paste into Claude Code. That installs the builder.

1
Turn on code execution. Settings, then Capabilities (needs a plan with Skills).
2
Download the builder:
skillc builder ↓
3
Upload it. Settings, then Capabilities, then Skills, then drop the zip in.
