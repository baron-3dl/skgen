self-building skills

Ship a skill that rebuilds itself
on whoever gets it.

A skill that works great for you usually comes out worse for someone else, because while you were getting it right it quietly leaned on your setup: your saved instructions, your memory, your tools. skillc ships a skill as one self-building file that rebuilds itself on the receiver and checks its own output against examples you approved. It either reproduces your quality, or it says plainly that it could not.

Install View on GitHub

The problem

A finished SKILL.md is a binary with undeclared dependencies.

As you develop a skill, it picks up implicit dependencies on your context. On your machine they resolve and it works. On anyone else's they resolve to something different or to nothing, so the output drifts. Nothing fails loud like a missing library. It just quietly produces the wrong thing. The fix is to ship source that rebuilds in place, and to declare every dependency on purpose: carry the ones that define the output, bind the ones that are genuinely local.

How it works

Like a self-extracting archive.

The file you send carries two things fused together: the material that defines good output, and the recipe that rebuilds the skill from it. The first time the receiver uses it, before it answers anything, it runs that recipe:

  1. Resolve the binds. Find each local tool the skill needs. If a required one is missing, stop and name it. Never guess.
  2. Rebuild. Write the instructions that reproduce the author's build examples on this model, in this session, overriding any local setting that fights the skill.
  3. Acceptance test. Run held-back examples it never saw while rebuilding, and score the honest transfer.
  4. Report, then run. State one line (built / honest-failure / cannot-build), then answer, checking every answer against the rules before sending.

Install

Claude Code

Add the marketplace

One line to add it, one to install the builder plus the example skills.

/plugin marketplace add baron-3dl/skillc /plugin install skillc@skillc
claude.ai (web)

Download and upload

Turn on code execution in Settings, then Capabilities. Download a skill's zip from the latest release and upload it under Settings, then Capabilities, then Skills.

# grab voice.zip, explain-plainly.zip, # commit-message.zip, or skillc.zip github.com/baron-3dl/skillc/releases/latest

Use it

Author

Turn a working skill into a portable one

In Claude Code or in a claude.ai chat (with the builder installed), point it at the skill you have been iterating:

build a self-building skill from <your skill>

It runs your skill, you approve the good outputs, and it writes one self-building file plus a ready-to-share zip.

Share

Send one file

That is the whole artifact. Drop it in a skills folder, upload it on claude.ai, or push it to a GitHub repo with a marketplace manifest so others install it with one command. It rebuilds and grades itself wherever it lands.

Provenance

Every commit is timestamped on Bitcoin.

Each push to main is anchored via OpenTimestamps, a forensically valid, independently verifiable proof of existence. The trust root is one small recipe seed you verify by reading, not by running. That is the whole inventorship claim.
